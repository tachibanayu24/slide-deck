# slide-deck: Claude Code Plugin for Slide Generation

## 何を作るか

Claude Codeプラグインとして、**LLMが高品質なプレゼンテーションを生成するためのフレームワーク**を作る。

## 背景・動機

- デライト・ベンチャーズ社内で、HTML + CSS + Web Componentsによるスライドシステムを運用してきた
- Claude Codeでスライドを生成すると実用レベルのものが出る。ただし毎回CSSを手書きする部分（グラフ、図解等）が非効率
- 「LLMにスライドを作らせる」ワークフローをplugin化して公開したい

## コアとなる設計思想

1. **HTMLが一級市民**: Markdown→HTML変換ではなく、HTML/CSS/JS/SVGをそのまま書く。表現の制約ゼロ
2. **Web Componentsは省略記法**: 共通パターンを短く書けるが、いつでも生HTMLに落とせる。ロックインしない
3. **PDFエクスポート**: ブラウザの印刷機能だけで完結。A4横、余白なし。外部ツール不要
4. **LLMフレンドリー**: Claude Codeがプラグインのコンテキストだけで正しいスライドを生成できる。READMEを毎回読ませない

## 要求

### Must Have

- プラグインを入れたClaude Codeが、指示に応じて高品質なHTMLスライドを生成できる
- PDF出力が崩れない（`print-color-adjust: exact` 等をデフォルト適用）
- `file://` で開けるならそうしたい
- データ可視化コンポーネント（棒グラフ、横棒、ドーナツ等）を属性だけで書ける

### Should Have

- レイアウトプリセット（左テキスト+右図解、全面図解、2×2マトリクス等）
- Mermaid.js統合（フロー図、シーケンス図をスライド内に描画）
- スラッシュコマンド体系

### Could Have

- AI画像生成連携（Gemini等でスライド内の概念図を生成）
- テーマ切り替え（ダーク、コーポレートカラー対応等）
- アニメーション/トランジション

## アイデア・メモ

### 図解の表現力を広げる

現状の課題: 概念図やフロー図を作るとき、CSSのインラインスタイルを大量に手書きしている。LLMは書けるが冗長で、PDF出力で壊れることがある

考えられるアプローチ:

- `<s-chart>` コンポーネント: `data` 属性にJSON/CSVを渡すとSVGで描画。印刷対応が楽
- `<s-mermaid>`: Mermaid.jsをラップ。SVGレンダリングなのでPDF出力と相性良い
- `<s-diagram>`: もっと自由度の高い図解。LLMがSVGを直接生成するのを前提にした枠組み
- 右半分にGemini画像生成の結果を配置する、みたいなレイアウトプリセット

### プラグインの形

Claude Codeプラグインとして実装する場合:

- スキル定義（`/slide` コマンド）でスライド生成のプロンプトとコンテキストを提供
- フレームワークのリファレンス（コンポーネント一覧、属性、使い分け）をプラグインコンテキストに含める
- LLMが「どのコンポーネントをいつ使うか」を判断できるガイドラインを組み込む

### 品質の担保

LLMが生成するスライドの品質を上げるために:

- レイアウトプリセットで「スライドに詰め込みすぎ」を防ぐ（A4横の物理制約を意識させる）
- `<s-chart>` 等でデータ可視化を属性化すれば、CSSの手書きミスによるPDF崩れがなくなる
- `examples/` にパターン別のサンプルを置き、few-shot的に参照させる

### 既存実装からの学び

元のシステム（`delight-ai/docs/presentations/common/`）で学んだこと:

- デザイントークン（CSS変数）で色を統一するのは非常に有効
- `<s-deck>` によるページ番号・PDFボタン自動付与は体験が良い
- `connectedCallback` の冪等性に注意が必要（親が子を移動すると再実行される）
- Lucide Iconsとの連携は便利だが、CDN依存になる
- `@media print` と `@page { size: A4 landscape; margin: 0; }` でPDF出力はほぼ完璧に動く

## 配布形態の候補

| 方式                       | 特徴                                                                       |
| -------------------------- | -------------------------------------------------------------------------- |
| npm + CDN (unpkg/jsDelivr) | `<script src="https://unpkg.com/slide-deck@1/...">` で即使用。最もシンプル |
| Claude Code Plugin (npm)   | `/slide` コマンドでスライド生成。フレームワーク自体もpluginが配置          |
| 両方                       | フレームワーク = npm、Claude Code連携 = plugin。分離が綺麗                 |
